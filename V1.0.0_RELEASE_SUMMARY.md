## âš ï¸ Critical Operations

### Document Posting
`post_document` is a **critical operation** that:
- Finalizes draft documents in the ledger
- Makes documents affect financial reports
- Makes documents non-editable
- **Cannot be undone** (except through reversal)

**Always verify document details before posting!**

### Document Reversal
`reverse_document` is an **important operation** that:
- Creates reversing entries for posted documents
- Affects ledger balances
- Creates permanent audit trail
- Should include reversal reason

**Use only for correcting genuine errors!**

- Maximum 500 results per query
- No pagination support (yet)
- Some filters are client-side
- Rate limit of 1500 requests per 5 minutes

### Workflow Limitations
- Document posting is irreversible
- Approval workflows depend on iplicit configuration
- Reversal creates new documents (doesn't delete)
